## 2. Panoramic snapshot — opencode-agent-memory

- **Pitch / philosophy**: persistent, **self-editable** memory across
  sessions. README: *"the markdown files on disk are shared state that
  every OpenCode session can read and write"*; *"all fields have
  defaults for graceful degradation, but `description` is essential."*
  Adapts **Letta's shared-memory-block** pattern.
- **Storage**: markdown + YAML frontmatter. Global blocks at
  `~/.config/opencode/memory/*.md`; project blocks at `.opencode/memory/
  *.md` (auto-gitignored).
- **Block frontmatter**: `label`, `description` (drives agent usage),
  `limit` (char cap, default 5000), `read_only`.
- **Three default blocks**: `persona` (global), `human` (global, user
  prefs), `project` (project-scoped, codebase knowledge).
- **Agent interface — dedicated tools**: `memory_list`, `memory_set`
  (overwrite whole block), `memory_replace` (substring edit). Optional
  journal: `journal_write` (append), `journal_search` (local
  all-MiniLM-L6-v2 embeddings — "no data leaves your machine"),
  `journal_read`.
- **Lifecycle**: blocks are injected into the **system prompt**, always
  in-context, surviving compaction without explicit load/save.
- **Scope**: global / project / session. **No role/identity scope.**
- Self-describes as *"Experimental"*; differentiates from OpenCode's
  `AGENTS.md` rules by *"structure (scoped blocks with metadata and
  size limits), dedicated tools for memory operations"* — explicitly
  positioned as complementary, not a replacement.

## 3. Panoramic snapshot — opencode-memory + OpenClaw Dreaming

### opencode-memory (chriswritescode-dev)

- **Storage**: **SQLite + local vector embeddings** (all-MiniLM-L6-v2;
  optional OpenAI / Voyage). `~/.local/share/opencode/memory/`.
- **Scopes (type, not role)**: `convention` / `decision` / `context`.
- **Semantic search**: `distanceThreshold` (default 0.5), 30 s result
  cache; matched memories injected into the user message via a
  `<project-memory>` block (2000-token budget).
- **Auto-dedup**: exact-match + semantic-similarity (threshold 0.25,
  clamped 0.05–0.40) + embedding-dimension-mismatch detection.
- **Compaction-prompt injection**: intercepts compaction, extracts
  conventions/decisions, injects them *into the compaction summary
  prompt* (4000-token budget, priority trimming) so they survive.
- Hooks: `experimental.chat.messages.transform` (per-message inject +
  read-only enforcement for an "Architect" agent); custom compaction
  prompt; plan store (7-day TTL) + review store.

### OpenClaw — Dreaming (memory consolidation)

- Opt-in, **disabled by default**, *experimental*. Background sweep
  modelled on sleep: **Light → REM → Deep**.
  - **Light** — ingest recent signals/recall traces, **dedupe**, stage
    candidates.
  - **REM** — **pattern extraction** (recurring themes/entities) +
    **memory decay**: flag un-reinforced `MEMORY.md` entries for removal.
  - **Deep** — *only phase that writes* `MEMORY.md`; promotes a
    candidate only if it passes three gates: `minScore 0.8`,
    `minRecallCount 3`, `minUniqueQueries 3`.
- **6-signal weighted scorer**: Relevance 0.30, Frequency 0.24, Query
  diversity 0.15, Recency 0.15, Consolidation 0.10, Conceptual
  richness 0.06.
- **Reviewable before write**: `openclaw memory promote` dry-run +
  Dreams UI before `--apply`.

## 4. Four-axis comparison

### a) Design philosophy

| | All-Might | opencode-agent-memory | opencode-memory | OpenClaw Dreaming |
|---|---|---|---|---|
| Primary scope axis | **role** (personality) | type (persona/human/project) + global/project/session | **type** (convention/decision/context) | single `MEMORY.md` |
| Per-role isolation | **✓** | ✗ | ✗ | ✗ |
| Memory store | markdown/YAML + git mirror | **markdown + YAML** | **SQLite + vector** | markdown (`MEMORY.md`) + recall state |
| Write model | `/remember` (append) + `/reflect` (curate) | agent **self-edits blocks** via tools | agent-driven + auto-dedup | **automated** scored promotion + decay |
| Consolidation trigger | agent judgment (sentinel / session-end / user) | none (blocks edited inline) | none | **background sweep** |

**Three convergences, all vindicating All-Might:**
1. **Markdown-on-disk is now the majority position.** opencode-agent-
   memory and OpenClaw both keep human-readable markdown; only
   opencode-memory (11★) reaches for SQLite+vector — the *same outlier
   slot* claude-mem occupied on 2026-05-22. That makes the
   "No SQLite memory store" non-goal validated by a *fourth*
   independent project, with the database camp shrinking, not growing.
2. **Role isolation remains structurally unique to All-Might.** None of
   the three can say "stdcell_owner remembers X, pll_owner remembers Y,
   no bleed." Same widening-moat observation as SuperClaude+Serena
   (2026-05-25) and OpenViking #1942 (2026-05-22).
3. **The "audit, never trim" choice is now contrarian *and* correct.**
   OpenClaw Dreaming auto-*decays* un-reinforced `MEMORY.md` entries.
   All-Might's `l1_rewriter.py` deliberately does the opposite:
   *"Audit, never trim. Evicting bullets silently would hide work the
   user wanted persisted."* For a precious-domain memory (a rarely-
   recalled-but-critical PLL spec), auto-decay is data loss; the git
   mirror exists precisely because memory deletion is dangerous.

### b) Feature coverage — what All-Might already has

The striking result of this pass: **All-Might already ships the
mechanisms these plugins are built around.**

| Mechanism (competitor) | All-Might equivalent | Status |
|---|---|---|
| Letta `persona`/`human`/`project` blocks | `ROLE.md` (persona) + `MEMORY.md` user-prefs (human) + `understanding/` (project) — *with role isolation* | **Covered, +isolation** |
| Block `limit` (char cap) | `l1_rewriter.py` byte cap (`DEFAULT_MAX_BYTES=4096`) → `.l1-over-cap` sentinel → `/reflect §4` triage | **Covered, safer (audit-not-trim)** |
| Compaction-prompt injection (opencode-memory) | `experimental.session.compacting` injects the scope framing *into the compaction prompt*; `session.compacted` re-primes next message | **Covered (both directions)** |
| Semantic journal search | L3 `journal/` via SMAK vector search (`/recall`) | **Covered** |
| Recall-frequency promotion (Dreaming "Frequency") | `usage-logger` → `memory/usage.log`; `/reflect §6`: *"Topic recalled often → promote to L2; recalls returning 0 → knowledge gap"* | **Covered (agent judgment)** |
| Dedupe + pattern extraction (Dreaming Light/REM) | `/reflect §2–§5` (scope audit, distill duplicates, log insights) | **Covered (agent judgment)** |

The only competitor mechanisms All-Might lacks are ones it has
**deliberately rejected**: SQLite+vector store (non-goal), MCP tool
indirection (2026-05-18 Q6), and *automated* scored decay (the
"audit, never trim" / write-path-failure-tail line).

### c) Mechanism design — the two distinct ideas worth holding up

1. **Dedicated memory *tools* vs direct file edit** (opencode-agent-
   memory's `memory_set`/`memory_replace`). All-Might's agent edits
   `MEMORY.md` / `understanding/*.md` directly via the editor. For
   *capable* models this is equivalent (CLAUDE.md's "trust the model").
   For the *weak/air-gap* models All-Might targets, a constrained
   `memory_replace`-style tool *could* reduce mis-edits — but this is
   the **same unproven "weak model" case as 2026-05-25 Q3 (explicit
   Triggers)**, and it adds tool surface that must be dual-mirrored. See
   §5 Q1.

2. **Quantified recall signals** (Dreaming's Frequency / Query-diversity
   / RecallCount gates). This is the one idea that touches a *real* All-
   Might artifact: `/reflect §6` *instructs* the agent to act on
   "recalled often" and "0-result recalls", but `usage.log` records
   only `<iso> <cmd> invoked session=<id>` — **not the query text or the
   result count**. So §6's signals are inferred from session context,
   not computable from the log. Dreaming's contribution is the *insight*
   that these signals are quantifiable; the simplicity-preserving
   adoption is *better evidence for `/reflect`*, not *automation*. See
   §5 Q4 and §7.

### d) Implementation details

- **opencode-memory's SQLite+vector path carries a maintenance tail**:
  embedding-dimension-mismatch detection, reindex prompts, distance-
  threshold tuning, 3 embedding-provider branches. This is the same
  write-path/infra failure-class flagged for OpenViking (#2132/#2154,
  2026-05-22) and ReflexionMemory (2026-05-25 Q5). All-Might's
  "SMAK ingest async via marker-file closure, markdown source of truth"
  avoids the whole class.
- **opencode-agent-memory's `journal_search` uses the exact same
  embedder** (all-MiniLM-L6-v2) that SMAK can sit on — confirming
  All-Might's L3 stack is on the mainstream local-embedding path, not
  an exotic one.
- **Dreaming is shipped opt-in / disabled-by-default / dry-run-gated** —
  OpenClaw's own caution signals the scored-automation path is not yet
  trustworthy. That is itself evidence for keeping `/reflect` as
  agent-judgment rather than racing to automate it.

## 6. Investment verdict

**No new feature clears the simplicity bar today** — and this pass is
the strongest "validation, not challenge" result of the series:

1. The closest same-layer competitor (opencode-agent-memory) **re-uses
   All-Might's own storage philosophy** (markdown + YAML on disk,
   local-embedding journal search) and lands *without* role isolation —
   throwing All-Might's per-role memory into sharper relief.
2. Every mechanism the new cohort is built around — block size caps,
   compaction injection, recall-frequency promotion, dedupe/pattern
   extraction — **already exists in All-Might** (`l1_rewriter.py`,
   `experimental.session.compacting`, `usage-logger`+`/reflect §6`),
   mostly in a safer, agent-judgment form.
3. The two mechanisms All-Might lacks are ones it **deliberately
   rejected** (SQLite/vector store, automated scored decay), and this
   pass adds fresh evidence for both rejections (opencode-memory is the
   lone 11★ SQLite outlier; OpenClaw ships Dreaming disabled-by-default).

**Concrete actions: none required.** The single candidate worth
tracking (Q5 / §7) stays deferred behind a user-experienced trigger,
consistent with every prior pass.

## 7. Feature rating — the one candidate worth rating (Q5)

**Enrich `usage.log` with `/recall` query text + result count so
`/reflect §6` is log-computable**, informed by OpenClaw Dreaming's
quantified recall signals.

| Dimension | Assessment |
|---|---|
| Problem reality | **Latent, unobserved.** `/reflect §6` already *instructs* acting on "recalled often" / "0-result" signals, but `usage.log`'s line format (`<iso> <cmd> invoked session=<id>`) can't supply them — the agent infers from context. No user has reported §6 as un-actionable. |
| Simplicity cost | **Low-mechanism, non-trivial policy.** ~15 LOC extending the existing `usage-logger` plugin + its Python mirror (no new surface). *But* it writes query text into an **unreviewed** log, a real PII cost given All-Might's PII-review discipline in `/one-for-all`; needs redaction. |
| Upside if real | Moderate — `/reflect §6` becomes evidence-driven instead of memory-from-context; folds the deferred 2026-05-22 Q6 `recall --trace` into the same change. |
| Reversibility | Good — log-format-only, additive. |
| **Verdict** | **Defer, do not build now. Rating 2/5 for investment.** Re-open trigger: *first user report that `/reflect §6` produces noise OR misses an obvious gap* — and build it together with `recall --trace`, with query-text redaction. Until then `/reflect`'s agent-judgment loop is the simpler, safer mechanism. |

Consistent with every prior pass: a candidate clears the bar only when
it solves a *real, experienced* problem that no existing mechanism
covers. Q5 has an existing mechanism (agent reads `usage.log` + context)
and an unproven problem; it earns a watch-flag, not a build.

## 8. Lessons captured for next reconnaissance

- **When the new cohort's headline features are already in your tree,
  that's the strongest validation there is.** This pass found *zero*
  mechanisms in opencode-agent-memory / opencode-memory / Dreaming that
  All-Might doesn't already have (or deliberately reject). Log the
  coverage table; it's the antidote to feature-envy.
- **"Audit, never trim" aged into a differentiator.** A safety choice
  made for L1 cap handling now distinguishes All-Might from a 373k-star
  project's flagship feature (Dreaming's auto-decay). Conservative
  data-handling choices compound into moats.
- **The database camp is shrinking.** Markdown-on-disk: All-Might,
  Serena, opencode-agent-memory, OpenClaw. SQLite/vector:
  claude-mem (suspicious stars), opencode-memory (11★). The "human-
  readable not database" non-goal is now the clear majority.
- **Distinguish the *insight* from the *implementation*.** OpenClaw
  Dreaming's useful contribution isn't its scorer — it's the reminder
  that recall frequency/diversity are *quantifiable*. Extract the
  signal idea (better `/reflect` evidence), discard the automation
  (scored decay). Same discipline as 2026-05-25's "different mechanism
  ≠ competing mechanism".
- **Same-layer beats adjacent-layer for comparison.** Prior memory
  passes (mem0/letta/honcho) were "the wrong axis" because they sit
  *below* All-Might. The OpenCode-native plugin cohort sits at the
  *same* layer — a much sharper mirror. Prefer same-layer comparators
  going forward.
